# Contribution Guidelines

Thank you for your interest in contributing to the IfcConstructionManagerProxy laydown planner! We appreciate your help and hope that this document will guide you in contributing effectively.

## Getting Started

1. **Fork the Repository**: Start by forking the repository on GitHub.
2. **Clone your Fork**: Clone your forked repository to your local machine.
   ```bash
   git clone https://github.com/your-username/laydown-planner.git
   ```
3. **Create a New Branch**: Itâ€™s a good practice to create a new branch for your work. Use the following command:
   ```bash
   git checkout -b your-feature-branch
   ```

## Making Changes

1. Make your changes in your feature branch.
2. **Test Your Changes**: Ensure to test your changes thoroughly. 
3. **Commit Your Changes**: Use clear, concise commit messages.
   ```bash
   git commit -m "Add your descriptive message here"
   ```

## Push Your Changes

Push your changes to your forked repository:
```bash
git push origin your-feature-branch
```

## Create a Pull Request

1. Go to the original repository where you want to merge your changes.
2. Click on the **Compare & pull request** button.
3. Provide a clear description of your changes and submit your pull request.

## Development Workflow

1. **Keep Your Fork Updated**: Synchronize your fork with the upstream repository regularly.
   ```bash
   git fetch upstream
   git checkout main
   git merge upstream/main
   ```
2. **Work on Your Changes**: Ensure to still work in a feature branch as described above.
3. **Review and Address Feedback**: Be responsive to code reviews and make necessary adjustments.

Thank you for contributing! We look forward to your input and appreciate your efforts to improve the project!
